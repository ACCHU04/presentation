1What is Recursion?

Recursion is when a function calls itself to solve smaller instances of the same problem. It needs a base case to stop, otherwise it goes infinite.

2Why Use Recursion?

  • Cleaner code for problems that split into similar subproblems.
  • Natural for trees, graphs, combinations, and divide-and-conquer algorithms.
  • Makes some algorithms (like DFS, permutations) straightforward.

4Basic Example — Factorial (Python)

Factorial recursively: n! = n × (n−1)! with 0! = 1.

# Factorial (Python)
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)

print(factorial(5))  # 120

5Fibonacci (Recursive Example)

Recursive Fibonacci splits into two subproblems: fib(n−1) and fib(n−2).

# Fibonacci (Python)
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

print([fib(i) for i in range(8)])  # [0,1,1,2,3,5,8,13]

9Advantages & Disadvantages

Advantages
  • Cleaner, often shorter code.
  • Natural fit for tree/graph traversal and divide-and-conquer.
  • Easier to reason about some algorithms (e.g., backtracking).
Disadvantages
  • Uses stack memory — risk of stack overflow.
  • Sometimes slower than iterative versions.
  • Requires careful base-case design to avoid infinite recursion.
